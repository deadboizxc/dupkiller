# dupkiller

Высокопроизводительный поиск дублирующихся файлов с многоядерным хешированием,
поддержкой возобновления прерванных сканирований и потоковым выводом для наборов
данных любого размера.

## Возможности

- **Быстрое хеширование** — BLAKE3 (определяется автоматически) с fallback на
  blake2b; 3-точечное сэмплирование (начало + середина + конец, 512 КБ каждый)
  для файлов > 50 МБ; адаптивный размер блока 256 КБ – 16 МБ
- **Многоядерность** — ThreadPoolExecutor для I/O-задач (частичное хеширование),
  ProcessPoolExecutor для CPU-задач (полное хеширование); `BoundedExecutor` с
  семафором предотвращает переполнение очереди и OOM при 1 M+ кандидатах
- **Поддержка HDD** — чтение файлов в порядке inode, сниженный параллелизм,
  `--ionice` (idle-приоритет I/O); автоопределение через `/sys/block/` на Linux
- **Возобновление** — состояние хранится в SQLite-сессии с PID-блокировкой и
  отслеживанием стадий; прерванное сканирование продолжается с той стадии, на
  которой остановилось
- **Безопасность при 1 ТБ+** — все итераторы используют keyset-пагинацию;
  потоковый вывод и потоковое удаление удерживают потребление памяти на уровне O(1)
- **Обнаружение жёстких ссылок** — файлы с одинаковой парой `(device, inode)`
  идентифицируются и исключаются из подсчёта дублей
- **Богатый CLI** — прогресс-бары в реальном времени, таблицы статистики, дельта
  между сканированиями, интерактивное удаление по группам, пять форматов экспорта
- **100% покрытие тестами** — юнит- и интеграционные тесты для всех модулей

## Требования

- Python 3.11+
- `click >= 8.1`
- `rich >= 13.0`
- `blake3` (опционально, ~5× быстрее): `pip install blake3`

## Установка

```bash
pip install dupkiller
```

Или из исходников:

```bash
git clone https://github.com/example/dupkiller
cd dupkiller
pip install -e .
```

## Быстрый старт

```bash
# Сканирование одной или нескольких директорий
dupkiller scan ~/Downloads ~/Documents

# Список групп дублей
dupkiller list

# Сводная статистика
dupkiller stats

# Dry-run: показать, что будет удалено, без реального удаления
dupkiller delete --keep newest --dry-run

# Удаление с подтверждением
dupkiller delete --keep newest --yes

# Интерактивное удаление: подтверждение для каждой группы
dupkiller delete --keep newest --interactive

# Экспорт в CSV
dupkiller export --format csv --output duplicates.csv
```

## Команды

### `scan`

Рекурсивное сканирование одного или нескольких каталогов с сохранением
результатов в базу данных.

```
dupkiller scan [OPTIONS] PATH [PATH ...]

Параметры:
  -t, --threads N         Потоков I/O для частичного хеширования  [авто: 2 HDD / 16 SSD]
  -p, --processes N       Процессов CPU для полного хеширования   [умолч: cpu_count()]
  --hdd / --ssd           Принудительный режим диска (автоопределение на Linux)
  --max-throughput RATE   Ограничение скорости чтения, напр. 50MB или 200KB
  --ionice                Запуск с idle-приоритетом I/O (Linux; требует ionice)
  --min-size N            Пропускать файлы меньше N байт           [умолч: 1]
  --max-size N            Пропускать файлы больше N байт
  --exclude PATTERN       Glob-шаблон для исключения (повторяемый)
  --follow-symlinks       Следовать символическим ссылкам
  --resume                Продолжить последнее прерванное сканирование
  --output-dir DIR        Директория для duplicates.jsonl и scan.log
  --db PATH               Путь к SQLite базе данных               [умолч: ~/.dupkiller/cache.db]
```

### `list`

Показать группы дублей из последнего сканирования.

```
dupkiller list [OPTIONS]

Параметры:
  --scan-id ID    Показать результаты конкретного сканирования
  --min-size N    Фильтр: файлы не меньше N байт
  --limit N       Максимальное количество групп для отображения   [умолч: 50]
```

### `stats`

Вывод сводной статистики; сравнение двух сканирований для выявления изменений.

```
dupkiller stats [OPTIONS]

Параметры:
  --scan-id ID              Статистика конкретного сканирования
  --compare-with ID         Дельта относительно другого сканирования
```

### `delete`

Удаление файлов-дублей; одна копия из каждой группы сохраняется согласно
выбранной стратегии.

```
dupkiller delete [OPTIONS]

Параметры:
  --keep STRATEGY   Что оставить: newest | oldest | first | shortest | longest
                    [умолч: newest]
  --dry-run         Показать, что будет удалено, без реального удаления
  --yes             Пропустить общее подтверждение перед удалением
  --interactive     Подтверждать каждую группу дублей отдельно
  --scan-id ID      Работать с конкретным сканированием
```

### `export`

Экспорт групп дублей в файл.

```
dupkiller export [OPTIONS]

Параметры:
  --format FMT    json | csv | txt | jsonl | html  [умолч: json]
  --output PATH   Запись в файл вместо stdout
  --scan-id ID    Экспорт конкретного сканирования
```

### `cache clean`

Удаление устаревших записей кеша для файлов, которых больше нет на диске.

```
dupkiller cache clean [--db PATH]
```

### `cache stats`

Вывод размера базы данных и количества строк в таблицах.

```
dupkiller cache stats [--db PATH]
```

## Стратегии сохранения (`--keep`)

| Стратегия  | Описание                                            |
|------------|-----------------------------------------------------|
| `newest`   | Сохранить файл с наиболее поздним временем изменения |
| `oldest`   | Сохранить файл с наиболее ранним временем изменения  |
| `first`    | Сохранить лексикографически первый путь             |
| `shortest` | Сохранить путь с наименьшим числом символов         |
| `longest`  | Сохранить путь с наибольшим числом символов         |

## Переменные окружения

| Переменная     | По умолчанию              | Описание                    |
|----------------|---------------------------|-----------------------------|
| `DUPKILLER_DB` | `~/.dupkiller/cache.db`   | Путь к SQLite базе данных   |

## Архитектура

```
CLI (click)
  └── run_pipeline()
        ├── Фаза 1 — scan_files()        ThreadPoolExecutor, фильтры исключений
        ├── Фаза 2 — частичное хеш.      ThreadPoolExecutor, LargeFileSemaphore
        ├── Фаза 3 — полное хеш.         ProcessPoolExecutor, BoundedExecutor
        └── Фаза 4 — вывод групп         OutputHandler → duplicates.jsonl
HashCache   (SQLite WAL, batch insert, keyset-пагинация)
ScanSession (PID-блокировка, стейт-машина стадий, resume)
```

Все четыре фазы сохраняются в SQLite-чекпоинте. Сканирование, прерванное на
любой стадии, продолжается с той же стадии без повторной обработки уже
обработанных файлов.

## Производительность

| Сценарий                        | Типичная производительность   |
|---------------------------------|-------------------------------|
| SSD + BLAKE3, полное хеш.       | ~3 ГБ/с                       |
| SSD + blake2b, полное хеш.      | ~700 МБ/с                     |
| HDD, чтение в порядке inode     | ~100–150 МБ/с                 |
| Prehash (файлы > 50 МБ)         | экономит до 99 % I/O          |

Советы:
- Установите `blake3` для максимальной скорости: `pip install blake3`
- На HDD используйте `--hdd` (или полагайтесь на автоопределение) для
  inode-упорядоченных чтений
- Используйте `--max-throughput` для ограничения нагрузки на загруженном сервере
- Для сетевых файловых систем рекомендуется `--processes 1` во избежание
  накладных расходов fork на NFS
